# ИС «НефтеЗакупки» — система анализа рынка и оптимизации закупок МТР

Информационная система для управления закупками материально-технических ресурсов (МТР) нефтедобывающего предприятия.

**Стек:** Django 4.2 + DRF · React 18 + Vite · PostgreSQL 16 · Nginx · Docker Compose

---

## Быстрый запуск через Docker Desktop

### Что нужно установить

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) для Windows

### Шаг 1 — Открыть терминал в папке проекта

Откройте папку `oil_procurement` в проводнике.  
Кликните правой кнопкой мыши на пустом месте → **«Открыть в терминале»** (или PowerShell).

### Шаг 2 — Собрать и запустить все контейнеры

```bash
docker compose up --build -d
```

Эта команда:
1. Соберёт образ **backend** (Django + Gunicorn)
2. Соберёт образ **frontend** (React → собирается через Node 20, отдаётся через Nginx)
3. Запустит **PostgreSQL** (база данных)
4. Запустит **Nginx** (обратный прокси на порту 80)
5. Автоматически выполнит миграции и загрузит демо-данные

Первый запуск занимает **3–7 минут** (скачивание образов + сборка React).

### Шаг 3 — Открыть сайт

Перейдите в браузере: **http://localhost:8080**

## Управление через Docker Desktop (графический интерфейс)

После запуска `docker compose up` откройте **Docker Desktop**:

| Что нажать | Что делает |
|---|---|
| Вкладка **Containers** | Показывает все 4 контейнера: `oil_nginx`, `oil_backend`, `oil_frontend`, `oil_db` |
| Кнопка ▶ (Play) рядом с контейнером | Запускает остановленный контейнер |
| Кнопка ⏹ (Stop) | Останавливает контейнер |
| Кнопка 🗑 (Delete) | Удаляет контейнер (данные в volumes сохраняются) |
| Кнопка **Logs** (значок листа) | Показывает логи контейнера в реальном времени |
| Кнопка **Terminal** (>_) рядом с `oil_backend` | Открывает bash внутри контейнера Django |
| Вкладка **Volumes** | Управление томами с данными БД и статикой |

### Как перезапустить проект целиком

В Docker Desktop → вкладка **Containers** → найдите строку с именем проекта → кнопка **Stop** → затем **Start**.

Или через терминал:

```bash
# Остановить
docker compose stop

# Запустить снова
docker compose start

# Остановить и удалить контейнеры (данные в volumes сохранятся)
docker compose down

# Пересобрать после изменений в коде
docker compose up --build -d
```

## Функционал системы

### Каталог МТР
- Справочник материалов с кодом МТР, техническими характеристиками, категорией и критичностью
- Складские остатки на карточке каждого материала
- График динамики цен по поставщикам (Recharts)

### Складской учёт
- Учёт остатков по местам хранения: всего / зарезервировано / свободно
- Подсветка позиций ниже минимального запаса

### Заявки на закупку (Workflow)
```
Черновик → На рассмотрении → Утверждена → Заказ размещён → Исполнена
                           ↘ Отклонена → (возврат в Черновик)
```
- Инициатор создаёт заявку и добавляет позиции МТР
- Закупщик анализирует КП и выбирает лучшего поставщика
- Руководитель утверждает или отклоняет

### Анализ рынка — сравнительная таблица КП
Каждое ценовое предложение получает балл по трём критериям:

| Критерий | Плановая | Аварийная |
|---|---|---|
| Цена | 60% | 20% |
| Срок поставки | 20% | 70% |
| Надёжность поставщика | 20% | 10% |

### Аналитика
- **Дашборд** — сводная статистика, пирог заявок по статусам, топ материалов
- **Динамика цен** — линейный график по выбранному материалу в разрезе поставщиков
- **Отчёт по дефициту** — позиции заявок к закупке + МТР ниже минимума
- **Сводные отчёты** — заявки по месяцам, заказы по статусам, топ поставщиков

### Ролевая модель

| Роль | Доступ |
|---|---|
| Инициатор | Создание заявок, каталог |
| Закупщик | Всё + КП, поставщики, заказы |
| Аналитик | Каталог, склад, аналитика, КП |
| Руководитель | Утверждение заявок, аналитика |
| Администратор | Полный доступ + управление пользователями + журнал операций |

## Полезные команды

```bash
# Просмотр логов в реальном времени
docker compose logs -f

# Логи только одного сервиса
docker compose logs -f backend

# Сбросить БД и загрузить демо-данные заново
docker compose exec backend python manage.py flush --noinput
docker compose exec backend python manage.py seed_data

# Запустить Django shell
docker compose exec backend python manage.py shell

# Полный сброс (удаляет контейнеры И тома с данными!)
docker compose down -v
docker compose up --build -d
```
